## 代码提交流程

### clone 仓库

```bash
git clone 
```
### 创建分支

第一步，保证main分支是最新的    

```bash
git pull origin master
```

第二步，创建分支并跳转
```bash
git checkout -b feature/<name>-<date>-<description>
```

例如：

```bash
git checkout -b feature/Kxh-20250508-add-dev-doc
```

或者先创建分支

```bash
git branch feature/Kxh-20250508-add-dev-doc
```

然后跳转    

```bash
git checkout feature/Kxh-20250508-add-dev-doc
```

除了feature 还可以是 fix/release/test/docs 等
- feature: 新功能git
- fix: 修复bug
- release: 发布
- test: 测试    
- docs: 文档


进行开发，并提交修改

```bash
git add .
git commit -m "feat(backend):add dev doc"
git push origin feature/kongxh-20250713-add-dev-doc
```

#### commit 信息格式

```bash
feat(backend):add dev doc
```
<任务分类>(<任务范围>):<任务描述>
任务分类与分支对应

- feat: feature
- fix: fix
- release: release
- test: test
- docs: docs

任务范围

- backend: 后端
- frontend: 前端
- docs: 文档
- test: 测试

任务描述：
描述干了什么就行

#### 在github合并分支
发起pull request
